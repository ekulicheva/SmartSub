# SmartSub — Умная подписка

Веб-приложение для управления личными подписками (Netflix, Spotify, ChatGPT и т.д.):
отслеживание расходов, напоминания о платежах, фильтрация и поиск.

Итоговая работа по веб-разработке. Стек: **.NET 10 Minimal API** (бэкенд) + **HTML/CSS/JS** (фронтенд), база данных **SQLite**, авторизация **JWT**.

## Структура проекта

```
SmartSub/
├── client/                 # Фронтенд
│   ├── index.html           # экран входа/регистрации + дашборд + модальное окно
│   ├── css/style.css        # стили, цветовая палитра, типографика
│   └── js/app.js            # авторизация, fetch-запросы к API, рендер, фильтры
│
└── server/                 # Бэкенд (SmartSub.Api)
    ├── Program.cs            # точка входа, конфигурация сервисов и middleware
    ├── appsettings.json      # строка подключения к БД, JWT-секрет
    ├── SmartSub.Api.csproj
    ├── Models/               # User, Subscription (+ enum-ы Period, Category)
    ├── Data/                 # EF Core DbContext
    ├── Dtos/                 # запросы/ответы API
    ├── Services/             # генерация JWT
    └── Endpoints/            # /api/auth/*, /api/subscriptions, /api/stats, /api/analytics, /api/profile/*
```

Фронтенд состоит из трёх страниц (переключаются без перезагрузки): **Дашборд** (статистика + список подписок), **Аналитика** (графики по категориям и прогноз, таблица всех платежей) и **Профиль** (данные аккаунта, настройка дней напоминания, смена пароля).

## Как запустить весь проект

### 1. Бэкенд

```bash
cd server
dotnet restore
dotnet run
```

Перед первым запуском замени заглушку `Jwt:Secret` в `server/appsettings.json` на свой длинный случайный ключ (32+ символов).

При первом старте база `smartsub.db` (SQLite) создаётся автоматически — миграции не требуются.

> **Важно:** если у тебя уже есть файл `smartsub.db` от запуска прошлой версии проекта (до добавления страниц «Аналитика» и «Профиль»), удали его перед повторным запуском — `EnsureCreated()` не обновляет схему существующей базы, а только создаёт новую с нуля.

Swagger-документация: `https://localhost:{port}/swagger` (порт будет в консоли при запуске).

### 2. Фронтенд

Просто открой `client/index.html` в браузере — бэкенд уже должен быть запущен.

Если бэкенд поднялся на адресе, отличном от `http://localhost:5000`, поменяй `window.API_BASE_URL` в `client/index.html` (в `<head>`).

На экране входа создай аккаунт через вкладку «Регистрация» — дальше вход будет автоматическим.

> Можно поднять и локальный сервер для фронтенда (удобно из-за CDN-шрифтов): `python3 -m http.server 8000` в папке `client`, затем открыть `http://localhost:8000`.

## Как это работает (логика клиент-сервер)

- При входе/регистрации фронтенд получает JWT-токен от `/api/auth/login` или `/api/auth/register` и сохраняет его в `localStorage`.
- Все запросы к подпискам (`/api/subscriptions`, `/api/stats`) идут с заголовком `Authorization: Bearer <token>`.
- Если токен истёк (ответ 401) — пользователя автоматически возвращают на экран входа.
- Поиск и фильтр по категории — реальные query-параметры (`?search=...&category=...`), обрабатываются на бэкенде.
- Категории и периодичность хранятся как enum-коды (`Music`, `Monthly` и т.д.), сериализуются как строки (`JsonStringEnumConverter`), на фронтенде переводятся в читаемый русский текст через словари в `app.js`.
- Каждый пользователь видит и может изменять только свои подписки — фильтрация по `UserId` из JWT-токена на каждом запросе.
- Пароли хранятся только как хеши (`PasswordHasher<User>`).
- Настройка «напомнить за N дней» в профиле задаёт значение по умолчанию для новых подписок; у уже существующих подписок своё значение, которое можно поменять индивидуально при редактировании.

## API: основные эндпоинты

| Метод  | URL                            | Auth | Описание                                  |
|--------|----------------------------------|------|---------------------------------------------|
| POST   | `/api/auth/register`             | —    | Регистрация: `{ email, password, displayName }` |
| POST   | `/api/auth/login`                | —    | Вход: `{ email, password }`                  |
| GET    | `/api/subscriptions`             | JWT  | Список подписок, `?search=&category=`        |
| POST   | `/api/subscriptions`             | JWT  | Создать подписку                             |
| PUT    | `/api/subscriptions/{id}`        | JWT  | Обновить подписку                            |
| DELETE | `/api/subscriptions/{id}`        | JWT  | Удалить подписку                             |
| GET    | `/api/stats`                     | JWT  | Расходы за месяц, прогноз на год, ближайший платёж |
| GET    | `/api/analytics`                 | JWT  | Разбивка расходов по категориям + прогноз на 6 месяцев |
| GET    | `/api/profile`                   | JWT  | Данные профиля: имя, email, дата регистрации, настройка напоминаний |
| PUT    | `/api/profile`                   | JWT  | Обновить имя и кол-во дней напоминания по умолчанию |
| POST   | `/api/profile/change-password`   | JWT  | Сменить пароль: `{ currentPassword, newPassword }` |

`period`: `Monthly` | `Quarterly` | `Yearly`
`category`: `Music` | `Movies` | `Work` | `Security` | `Cloud` | `Other`

## Возможные доработки

- Перейти с `EnsureCreated()` на полноценные EF Core миграции при изменении схемы БД.
- Фоновая задача (`IHostedService`) для реальных уведомлений о приближающихся платежах.
- Ограничить CORS конкретным доменом фронтенда вместо `AllowAnyOrigin` при продакшен-развёртывании.
